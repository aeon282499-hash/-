# KabuAI クローン — Claude Code 指示書

KabuAI（株価モメンタム分析アプリ）を **完全無料構成** で再現するための仕様書。
そのまま Claude Code に渡せるように書いています。末尾に「最初に貼り付ける指示文」あり。

## 0. 最初に読むべき「すべて無料」の現実

「全部タダ」は3つの壁があります。先に意思決定しておくと後がラク。

1. **株価データ** … 完全無料だと「遅延 or EOD（日足終値）」が前提。リアルタイムは無料では実質ムリ。
   - 一番現実的：**J-Quants 無料プラン**（あなたが既に使っている）。ただし無料は約12週間遅延。
   - 代替の無料EOD：Stooq（`.jp` 銘柄、CSVで日足取得可）、yfinance（`7203.T` 形式）。※両方とも再配布・利用規約に注意。
   - → **割り切り方針**：「データ日付」を画面に明示（KabuAI本家も `データ日付 2026年6月3日` と表示している）。リアルタイム風に見せない。

2. **AI機能（材料サマリー / AIリサーチ結果）** … LLMが要る。これは無料枠で回す。
   - **Google Gemini API 無料枠** か **Groq 無料枠（Llama系）** が現状いちばん「本当に無料」。低頻度ならこれで足りる。
   - もしくは **ローカル Ollama**（自分のPCで完結＝完全無料、ただし常時起動が必要）。
   - 大量銘柄を毎日回すと無料枠を超える → **バッチで1日1回だけ生成してJSONに保存**する設計にする。

3. **ホスティング** … これは無料で問題なし。
   - フロント（単一HTML）：**GitHub Pages**。
   - バッチ実行（毎日データ更新）：**GitHub Actions の cron**（無料枠で十分）。サーバー常駐すら不要。

> **結論**：「毎日1回 GitHub Actions が J-Quants 等からデータ取得 → 指標計算 → AI要約 → 静的JSON生成 → GitHub Pages が表示」。サーバー代0円、API代0円（無料枠内）で本家の95%は再現できる。リアルタイムPTSだけは妥協が必要。

## 1. 重要な前提・制約（ライセンス）

KabuAI本家は全画面に注意書きを出している：
> 「価格・チャートはAI等で整理・加工した参考表示です。リアルタイムの市場データ配信を目的としたものではありません。」

これはあなたが既に採用している「**生の株価を再配布せず、計算済みの分析値だけ出力する**」方針とまったく同じ。クローンでも厳守する：

- 生のOHLCVをそのままAPI公開・再配布しない。**計算後の指標（モメンタム指数・SR・ランク等）と、加工済みチャート画像/データのみ**を出す。
- J-Quants の商用デプロイ条件は JPX へ要確認（あなたの既知の宿題）。**公開前に必ず確認**。
- 全画面に免責（投資判断は自己責任 / 参考情報）を明記。

## 2. 技術スタック（既存プロジェクトに合わせる）

- **フロント**：単一HTML + バニラJS（あなたの既存テーマトラッカーと同じ）。チャートは Chart.js もしくは軽量な自前Canvas。
- **バックエンド/バッチ**：Python。データ取得（J-Quants V2）＋指標計算＋AI要約。
- **データ受け渡し**：バッチが `data/YYYYMMDD.json`（+ `latest.json`）を生成。フロントは fetch するだけ。
- **自動化**：GitHub Actions（平日夕方に cron 実行）。

- **主要指標カード**（各カード「タップで意味を確認」）：

- **警戒メモ**：例「天井圏に近く、過熱確認を優先」

### 3.4 KabuAIシグナル
- フィルタタブ：強買い集め / 強押し目 / 昇格 / 買い集め / 加速 / 反転 / 話題集中
- 各シグナルの説明 + `◯銘柄点灯` + 警戒/ショート等のバッジ
- 銘柄カード：銘柄名 / コード / 価格 / モメンタム指数 / RR / `発動日` / 発動日からの 5日・10日・1ヶ月パフォーマンス（未到来なら「経過待ち」）

### 3.5 PTSシグナル
- 仕様メタ：出来高1,000株以上 / 材料(決算・業績修正・TOB・自社株買い等を3行サマリー) / 対象=夜間PTS上昇TOP20・下落TOP20 / 更新17:05頃〜1時間ごと

- 一覧：順位 / 銘柄 / 変動率 / 材料バッジ（増配・監理銘柄・決算 等）

## 4. モメンタム指数の計算ロジック（動画で公開済み）

```
base       = SR × 18            # SR = シャープレシオ = リターン ÷ 値動きのブレ（実測 -2〜+5 に収まる）
power_adj  = clamp(出来高込みの押し上げ力, -3, +8)
momentum   = clamp(base + power_adj, 0, 100)
```

- ランク帯：S(80超) / A / B(40〜59) / C / D（D=失速・停滞、リバウンド余地）
- 補足：計算窓・係数・閾値は最適化で決める前提（同じ式でも母集団で結果が変わる、と本家も明記）
- 実装メモ：
  - SR は「期間リターン ÷ 期間ボラティリティ」。窓は20〜60営業日あたりで要調整。
  - POWER は出来高と値幅の合成（例：出来高前日比 × 当日値幅）を正規化して [-3,+8] にスケール。
  - RSI は標準14日。STAB は値動きの安定度（ボラの逆数系）。

## 5. AI機能の実装（無料枠で）

- 対象：PTSの「AIリサーチ結果（事実/要因）」、銘柄の警戒メモ、シグナル解説。
- 入力：TDnet適時開示の見出し・本文（無料で取得可）、指標値。
- 出力フォーマットを固定（JSONで `{ "fact": "...", "cause": "...", "stance": "警戒|中立|期待" }`）してLLMに投げる。
- **1日1回バッチで生成 → JSONに保存**（無料枠超過を防ぐ）。
- LLM：Gemini無料枠 / Groq無料枠 / ローカルOllama のいずれか。`.env` で切替できるよう抽象化。

## 6. 推奨ビルド順（Claude Codeに段階指示する）

1. データ取得層（J-Quants V2 → 銘柄リスト＋日足）＋ローカルJSONキャッシュ
2. 指標計算（SR / POWER / RSI / STAB / モメンタム指数 / ランク）

4. フロント：単一HTML、ホーム → ランキング → 銘柄詳細 → シグナル の順
5. チャート（ローソク＋指数オーバーレイ2軸）
6. AI要約バッチ（無料LLM）
7. GitHub Actions cron + GitHub Pages デプロイ
8. 免責・データ日付表示・ライセンス整理
